# 🧠 1. Recap: N+1 Problem

👉 1টা root query + প্রতিটা child এর জন্য আলাদা query

```
1 + N queries ❌
```

---

# 🚀 2. All Ways to Solve N+1 (Important 🔥)

---

# 🥇 1. DataLoader (Most Popular & Recommended)

👉 Already দেখেছো, but summary:

✔ batching
✔ caching
✔ async

```java
return dataLoader.load(user.getId());
```

👉 DB call:

```sql
SELECT * FROM post WHERE user_id IN (1,2,3)
```

✅ Best for:

* GraphQL field resolver
* Microservice (gRPC / REST)

---

# 🥈 2. JOIN Fetch (JPA / SQL level)

👉 DB level এ fix করা

```java
@Query("SELECT u FROM User u JOIN FETCH u.posts")
List<User> findUsersWithPosts();
```

✔ Single query:

```sql
SELECT * FROM user u JOIN post p ON u.id = p.user_id
```

✅ Best for:

* small/medium dataset
* simple relation

❌ Problem:

* over-fetching
* pagination issue

---

# 🥉 3. Batch Query Service Layer

👉 manually batch করা

```java
Map<Long, List<Post>> postsMap =
    postService.findPostsByUserIds(userIds);
```

✔ DataLoader ছাড়াও use করা যায়

---

# 🧩 4. GraphQL Projection / SelectionSet

👉 Client কী field চায় সেটা দেখে query optimize করা

```java
Set<String> fields = env.getSelectionSet()
                       .getFields()
                       .stream()
                       .map(f -> f.getName())
                       .collect(Collectors.toSet());
```

👉 based on fields:

```java
if(fields.contains("posts")) {
    // join fetch
}
```

✅ Smart optimization

---

# ⚡ 5. DTO / Projection Query

👉 Only required columns fetch

```java
public interface UserDTO {
    String getName();
}
```

✔ DB load কমে

---

# 🔥 6. Pre-fetch Strategy (Manual Optimization)

👉 আগে থেকেই data load

```java
List<Post> posts = postService.findAll();
Map<Long, List<Post>> grouped =
    posts.stream().collect(groupingBy(Post::getUserId));
```

✔ Resolver এ reuse

---

# 🌐 7. Microservice Batch API

👉 তোমার case (gRPC / REST)

Instead of:

```
getPosts(userId=1)
getPosts(userId=2)
```

👉 Use:

```
getPostsByUserIds([1,2,3])
```

✔ 1 network call only

---

# ⚠️ 8. Caching (Extra Optimization)

👉 Redis / local cache

```java
@Cacheable("posts")
```

✔ repeated call avoid
