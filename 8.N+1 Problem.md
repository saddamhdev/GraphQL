# 🧠 1. N+1 Problem কী?

ধরো তুমি এই query করছো:

```graphql
query {
  getUsers {
    name
    posts {
      title
    }
  }
}
```

---

## ❌ Without DataLoader (Problem)

👉 Execution flow:

```text
1 query → getUsers (1 DB call)

Then for each user:
   user1 → posts (1 DB call)
   user2 → posts (1 DB call)
   user3 → posts (1 DB call)
```

👉 Total:

```
1 + N query ❌
```

📌 যদি 100 user হয় → 101 query 😱

---

# 🔥 2. Why this happens?

কারণ GraphQL:

✔ field-by-field resolve করে
✔ `User.posts` আলাদা resolver call করে

---

# 🚀 3. Solution → DataLoader

👉 DataLoader করে:

✔ **Batching** → একসাথে query
✔ **Caching** → same request reuse

---

# 🧩 4. Concept (Simple)

Instead of:

```
getPostsByUserId(1)
getPostsByUserId(2)
getPostsByUserId(3)
```

👉 DataLoader করবে:

```
getPostsByUserIds([1,2,3]) ✅
```

# ⚙️ 5. Spring Boot Implementation

---

## ✅ Step 1: DataLoader define

```java
@Component
public class PostDataLoader {

    public DataLoader<Long, List<Post>> postLoader(PostService postService) {

        return DataLoader.newMappedDataLoader(userIds -> {

            // batch query
            Map<Long, List<Post>> postsMap =
                    postService.findPostsByUserIds(userIds);

            return CompletableFuture.completedFuture(postsMap);
        });
    }
}
```

---

## ✅ Step 2: Register DataLoader

```java
@Configuration
public class DataLoaderConfig {

    @Bean
    public DataLoaderRegistry dataLoaderRegistry(PostDataLoader loader) {

        DataLoaderRegistry registry = new DataLoaderRegistry();

        registry.register("postLoader", loader.postLoader());

        return registry;
    }
}
```

---

## ✅ Step 3: Use in Resolver

```java
@SchemaMapping(typeName = "User", field = "posts")
public CompletableFuture<List<Post>> posts(
        User user,
        DataFetchingEnvironment env) {

    DataLoader<Long, List<Post>> loader =
            env.getDataLoader("postLoader");

    return loader.load(user.getId());
}
```

# 🔥 6. Execution Flow (With DataLoader)

```text
getUsers → 1 query

collect all userIds → [1,2,3]

single batch query:
SELECT * FROM posts WHERE user_id IN (1,2,3)
```

👉 Total:

```
1 + 1 query ✅
```

# ⚠️ 9. Important Rules

✔ DataLoader per request scoped
✔ async return (`CompletableFuture`)
✔ batch method efficient হতে হবে

# 🎯 10. Interview Answer (Short)

👉 N+1 problem = multiple DB calls per parent entity
👉 DataLoader = batching + caching
👉 reduces queries from `1 + N → 1 + 1`
