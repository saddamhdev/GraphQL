GraphQL এ **`User` vs `[User]`** — এটা খুব basic কিন্তু খুব important concept 🔥
চল একদম clear করে বুঝি 👇

---

# 🧩 1. `User` (Single Object)

👉 মানে: **একটা User object return করবে**

```graphql
type Query {
  getUser(id: ID!): User
}
```

📌 Response:

```json
{
  "data": {
    "getUser": {
      "id": "1",
      "name": "Saddam"
    }
  }
}
```

✔ Use যখন:

* Single data দরকার
* ID দিয়ে fetch
* One-to-one relation

---

# 📦 2. `[User]` (List of Users)

👉 মানে: **multiple User list return করবে**

```graphql
type Query {
  getUsers: [User]
}
```

📌 Response:

```json
{
  "data": {
    "getUsers": [
      { "id": "1", "name": "Saddam" },
      { "id": "2", "name": "Rahim" }
    ]
  }
}
```

✔ Use যখন:

* List দরকার
* Multiple data
* One-to-many relation

---

# ⚠️ 3. Important: `!` (Non-null) combinations

GraphQL এ `[User]` এর সাথে `!` combine করলে meaning change হয় 👇

---

## 🔹 `[User]`

👉 List optional, element optional
✔ null হতে পারে
✔ ভিতরে null থাকতে পারে

---

## 🔹 `[User]!`

👉 List must exist
✔ null হবে না
❗ কিন্তু ভিতরে null থাকতে পারে

---

## 🔹 `[User!]`

👉 List optional
✔ কিন্তু list এর ভিতরে null allowed না

---

## 🔹 `[User!]!` ⭐ (Best practice)

👉 Listও null না + itemও null না

```graphql
type Query {
  getUsers: [User!]!
}
```

✔ Production এ এইটাই বেশি use হয়

---

# 🧠 4. Real Example (Relation)

```graphql
type User {
  id: ID!
  name: String!
  posts: [Post!]!
}
```

👉 মানে:

* User এর multiple Post আছে
* কোন Post null না

---

# 🎯 Summary (Interview Ready)

| Type       | Meaning                 |
| ---------- | ----------------------- |
| `User`     | Single object           |
| `[User]`   | List of objects         |
| `[User!]!` | Safe list (recommended) |

---

# 🚀 Pro Tip (Senior Level)

✔ Single → `User`
✔ List → `[User!]!`
✔ Naming → `user` vs `users`
