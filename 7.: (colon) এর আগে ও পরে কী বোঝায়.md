তুমি আসলে GraphQL এর এই syntax টা বুঝতে চাও 👉 **`:` (colon) এর আগে ও পরে কী বোঝায়** — খুব important concept 🔥

চল একদম clear করি 👇

---

# 🧠 Basic Rule

👉 **`name: Type`**

* 🔹 **Colon এর আগে → field / method নাম**
* 🔹 **Colon এর পরে → return type**

---

# 🧩 1. Simple Example

```graphql
type User {
  name: String
}
```

✔ Meaning:

* `name` = field name
* `String` = type (return করবে string)

---

# 🔍 2. Query Example

```graphql
type Query {
  getUser(id: ID!): User
}
```

✔ Breakdown:

* `getUser` → method নাম
* `(id: ID!)` → input parameter
* `User` → return type

👉 পুরো meaning:

> getUser function call করলে → User object return করবে

---

# ✍️ 3. Mutation Example

```graphql
type Mutation {
  createUser(name: String!): User
}
```

✔ Meaning:

* `createUser` = function
* `name: String!` = input
* `User` = return

---

# 📦 4. Input Type

```graphql
input CreateUserInput {
  name: String!
}
```

✔ Meaning:

* `name` = field
* `String!` = required string

---

# 🔁 5. Relation Example

```graphql
type User {
  posts: [Post]
}
```

✔ Meaning:

* `posts` = field name
* `[Post]` = list of Post return করবে

---

# ⚠️ 6. Important: Parameter এর ভিতরে colon

```graphql
getUser(id: ID!)
```

👉 এখানে:

* `id` = parameter name
* `ID!` = parameter type

---

# 🔥 Full Example (সব একসাথে)

```graphql
type Query {
  getUser(id: ID!): User
}
```

👉 Breakdown:

| Part    | Meaning       |
| ------- | ------------- |
| getUser | function name |
| id      | input name    |
| ID!     | input type    |
| User    | return type   |

---

# 🎯 Real-Life Analogy

👉 Java method এর মতো চিন্তা করো:

```java
User getUser(Long id)
```

GraphQL version:

```graphql
getUser(id: ID!): User
```

✔ Same concept 😄

---

# 🧠 Final Summary

👉 Colon এর আগে:

* field / function / parameter নাম

👉 Colon এর পরে:

* data type (কি return করবে / কি input নিবে)

---
