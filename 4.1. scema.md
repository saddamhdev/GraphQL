GraphQL schema হচ্ছে পুরো API-এর **blueprint / contract** — এখানে define করা থাকে client কী request করতে পারবে এবং server কীভাবে response দিবে।

চল সহজভাবে দেখি 👉 GraphQL schema তে কী কী থাকে:

---

# 🧩 1. **Types (Object Types)**

এগুলো হলো data structure

```graphql
type User {
  id: ID!
  name: String!
  email: String
}
```

📌 এখানে:

* `ID`, `String` = scalar type
* `!` = required (non-null)

---

# 🔍 2. **Query (Read Operation)**

Data fetch করার জন্য

```graphql
type Query {
  getUser(id: ID!): User
  getAllUsers: [User]
}
```

👉 Client call করবে:

```graphql
query {
  getUser(id: "1") {
    name
    email
  }
}
```

---

# ✍️ 3. **Mutation (Write Operation)**

Create / Update / Delete এর জন্য

```graphql
type Mutation {
  createUser(name: String!, email: String!): User
  deleteUser(id: ID!): Boolean
}
```

---

# 🔁 4. **Subscription (Realtime)**

Live data (WebSocket)

```graphql
type Subscription {
  userCreated: User
}
```

👉 Chat app / notification system এ use হয়

---

# 🧱 5. **Scalar Types**

Built-in:

* `Int`
* `Float`
* `String`
* `Boolean`
* `ID`

Custom scalarও করা যায়:

```graphql
scalar Date
```

---

# 🧩 6. **Input Type**

Mutation input clean রাখার জন্য

```graphql
input UserInput {
  name: String!
  email: String!
}
```

```graphql
type Mutation {
  createUser(input: UserInput): User
}
```

---

# 🔗 7. **Enum**

Fixed value list

```graphql
enum Role {
  ADMIN
  USER
}
```

---

# 🧬 8. **Interface**

Common structure

```graphql
interface Person {
  id: ID!
  name: String!
}
```

---

# 🧪 9. **Union Type**

Multiple possible return type

```graphql
union SearchResult = User | Product
```

---

# 📦 10. **Schema Definition**

Root define করে

```graphql
schema {
  query: Query
  mutation: Mutation
  subscription: Subscription
}
```

---

# 🔥 Real Example (Full Schema)

```graphql
type User {
  id: ID!
  name: String!
}

type Query {
  users: [User]
}

type Mutation {
  addUser(name: String!): User
}
```
