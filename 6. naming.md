GraphQL এ **naming convention খুবই important** — especially তুমি যেহেতু Spring Boot + microservice project করো, clean naming না হলে API mess হয়ে যাবে 😄

চল একদম practical rule গুলো দেখি 👇

---

# 🎯 1. General Naming Rules

✅ **PascalCase → Type, Enum, Interface**
✅ **camelCase → field, query, mutation**
✅ **UPPER_CASE → enum values**

---

# 🧩 2. Type Naming (PascalCase)

```graphql
type User {
  id: ID!
  name: String!
}
```

✔ Good:

* `User`
* `UserProfile`
* `OrderItem`

❌ Bad:

* `user`
* `user_data`

---

# 🔍 3. Query Naming (camelCase)

👉 Query = data fetch

```graphql
type Query {
  getUser(id: ID!): User
  listUsers: [User]
}
```

✔ Common pattern:

* `getUser`
* `listUsers`
* `findUserByEmail`

💡 Pro tip:
👉 REST এর মত `/users` না, meaningful action name দাও

---

# ✍️ 4. Mutation Naming (camelCase + action)

👉 Always verb দিয়ে শুরু

```graphql
type Mutation {
  createUser(input: CreateUserInput): User
  updateUser(id: ID!, input: UpdateUserInput): User
  deleteUser(id: ID!): Boolean
}
```

✔ Pattern:

* `createX`
* `updateX`
* `deleteX`

---

# 📦 5. Input Type Naming

👉 Always `Input` suffix use করো

```graphql
input CreateUserInput {
  name: String!
  email: String!
}
```

✔ Pattern:

* `CreateUserInput`
* `UpdateUserInput`

---

# 🔗 6. Field Naming (camelCase)

```graphql
type User {
  fullName: String
  createdAt: String
}
```

✔ Good:

* `fullName`
* `createdAt`

❌ Bad:

* `full_name`
* `CreatedAt`

---

# 🧬 7. Enum Naming

```graphql
enum UserRole {
  ADMIN
  USER
  SUPER_ADMIN
}
```

✔ Rules:

* Enum name → PascalCase
* Values → UPPER_CASE

---

# 🧪 8. Relation Naming

👉 meaningful name দাও

```graphql
type User {
  posts: [Post]
}
```

✔ Good:

* `posts`
* `author`

❌ Bad:

* `data1`
* `list`

---

# 🔥 9. Real Clean Example

```graphql
type User {
  id: ID!
  name: String!
  email: String!
  posts: [Post]
}

type Post {
  id: ID!
  title: String!
  author: User
}

input CreatePostInput {
  title: String!
  content: String!
  userId: ID!
}

type Query {
  getUser(id: ID!): User
  listPosts: [Post]
}

type Mutation {
  createPost(input: CreatePostInput): Post
}
```

---

# 🧠 10. Interview Ready Summary

👉 Naming rules:

* Type → `User` ✅
* Query → `getUser` ✅
* Mutation → `createUser` ✅
* Input → `CreateUserInput` ✅
* Field → `createdAt` ✅
* Enum → `ADMIN` ✅

---

# 🚀 Pro Level Tips (Senior Level)

✔ plural naming for list → `users`, `posts`
✔ avoid generic names (`data`, `info`)
✔ consistency maintain (সব query একই pattern follow করবে)
✔ REST → GraphQL mapping clean রাখো
