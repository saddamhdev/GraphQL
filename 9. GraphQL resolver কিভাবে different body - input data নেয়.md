# 🧠 1. Core Idea

👉 Resolver data নেয় ৩টা জায়গা থেকে:

1. **Arguments (input params)**
2. **Parent object (nested resolver)**
3. **Context / Environment (extra info)**

---

# 🧩 2. Case 1: Simple Argument (Single Input)

### Schema:

```graphql
type Query {
  getUser(id: ID!): User
}
```

### Resolver:

```java
@QueryMapping
public User getUser(@Argument Long id) {
    return userService.findById(id);
}
```

✔ এখানে:

* `id` → client থেকে আসছে
* `@Argument` → bind করছে

---

# 📦 3. Case 2: Multiple Arguments

### Schema:

```graphql
type Query {
  searchUser(name: String, age: Int): [User]
}
```

### Resolver:

```java
@QueryMapping
public List<User> searchUser(
        @Argument String name,
        @Argument Integer age) {

    return userService.search(name, age);
}
```

✔ Multiple field receive করা যায়

---

# 🔥 4. Case 3: Input Object (Best Practice)

### Schema:

```graphql
input CreateUserInput {
  name: String!
  email: String!
}
```

```graphql
type Mutation {
  createUser(input: CreateUserInput): User
}
```

---

### Resolver:

```java
@MutationMapping
public User createUser(@Argument CreateUserInput input) {
    return userService.create(input.getName(), input.getEmail());
}
```

✔ Clean & scalable
✔ production এ mostly এইটাই use হয়

---

# 🔁 5. Case 4: Nested Resolver (Parent Data)

### Schema:

```graphql
type User {
  posts: [Post]
}
```

---

### Resolver:

```java
@SchemaMapping(typeName = "User", field = "posts")
public List<Post> posts(User user) {
    return postService.findByUserId(user.getId());
}
```

✔ এখানে:

* `User user` → parent object
* client body থেকে না, previous resolver থেকে আসে

---

# ⚡ 6. Case 5: Context / Environment Data

👉 advanced use

```java
@QueryMapping
public User getUser(
        @Argument Long id,
        DataFetchingEnvironment env) {

    String userAgent = env.getGraphQlContext().get("userAgent");

    return userService.findById(id);
}
```

✔ Header / token / metadata access করা যায়

# 🧩 7. Case 6: Mixed Input (Real Use Case)

### Schema:

```graphql
type Mutation {
  createPost(userId: ID!, input: CreatePostInput): Post
}
```

---

### Resolver:

```java
@MutationMapping
public Post createPost(
        @Argument Long userId,
        @Argument CreatePostInput input) {

    return postService.create(userId, input);
}
```

✔ Multiple input combine করা যায়

# 🎯 9. Summary

| Input Source | Resolver এ কীভাবে আসে        |
| ------------ | ---------------------------- |
| Argument     | `@Argument`                  |
| Input object | `@Argument DTO`              |
| Parent data  | method parameter (User user) |
| Context      | `DataFetchingEnvironment`    |

✔ Token data → context এ রাখো
✔ Nested resolver → parent use করো
