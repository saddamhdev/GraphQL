# 🧩 1. GraphQL Schema Design (How to design)

## ✅ Step 1: Entity চিন্তা করো (Database like)

ধরো তোমার system:

* User
* Post

👉 তাহলে schema design শুরু হবে entity দিয়ে

```graphql
type User {
  id: ID!
  name: String!
  email: String!
}

type Post {
  id: ID!
  title: String!
  content: String!
  author: User
}
```

---

## ✅ Step 2: Relationship define করো

👉 SQL join এর মত

```graphql
type User {
  id: ID!
  name: String!
  posts: [Post]
}
```

---

## ✅ Step 3: Query design (READ)

👉 Client কী data চাইবে সেটা চিন্তা করো

```graphql
type Query {
  getUser(id: ID!): User
  getAllUsers: [User]
  getPosts: [Post]
}
```

---

## ✅ Step 4: Mutation design (WRITE)

👉 CRUD operation

```graphql
type Mutation {
  createUser(name: String!, email: String!): User
  createPost(title: String!, content: String!, userId: ID!): Post
}
```

---

## ✅ Step 5: Input type use করো (Best Practice)

👉 Clean API

```graphql
input CreateUserInput {
  name: String!
  email: String!
}

type Mutation {
  createUser(input: CreateUserInput): User
}
```

---

# 🎯 Schema Design Best Practices

✔ Small & reusable types
✔ Nested relation support
✔ Input আলাদা রাখো
✔ Over-fetching avoid করো
✔ Naming clear রাখো

# 🔍 2. Query লেখার নিয়ম (Syntax Rules)

## 🧠 Rule 1: Always field select করতে হবে

GraphQL এ তুমি কী data চাও সেটাই specify করতে হবে

```graphql
query {
  getUser(id: "1") {
    name
    email
  }
}
```

---

## 🧠 Rule 2: Nested query করা যায়

👉 SQL join এর alternative

```graphql
query {
  getUser(id: "1") {
    name
    posts {
      title
    }
  }
}
```

---

## 🧠 Rule 3: Arguments pass করা যায়

```graphql
query {
  getUser(id: "10") {
    name
  }
}
```

---

## 🧠 Rule 4: Alias ব্যবহার করা যায়

```graphql
query {
  user1: getUser(id: "1") {
    name
  }
  user2: getUser(id: "2") {
    name
  }
}
```

---

## 🧠 Rule 5: Variables use করা (Best Practice)

```graphql
query GetUser($id: ID!) {
  getUser(id: $id) {
    name
    email
  }
}
```

👉 Variables:

```json
{
  "id": "1"
}
```

---

## 🧠 Rule 6: Fragment reuse করা যায়

```graphql
fragment userFields on User {
  name
  email
}

query {
  getUser(id: "1") {
    ...userFields
  }
}
```
